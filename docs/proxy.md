# 正向代理 (Forward Proxy) vs 反向代理 (Reverse Proxy)

## 1. 正向代理 (Forward Proxy)

*   **关键词**：找梯子、保护客户端。

正向代理位于客户端（你）和原始服务器之间。为了从原始服务器取得内容，客户端向代理发送一个请求并指定目标，然后代理向原始服务器转交请求并将获得的内容返回给客户端。

*   **比喻**： 你想去一家不直接卖东西给你的商店（比如某些海外网站），你找了一个代购（代理服务器）。你告诉代购：“帮我买那个东西。”代购去买了以后交给你。

### 主要用途

1.  **科学上网**： 突破地理位置或防火墙限制。
2.  **隐藏身份**： 服务器只知道代理服务器来过，不知道真正发起请求的是你。
3.  **缓存与授权**： 公司内部网限制员工只能访问特定网站。

## 2. 反向代理 (Reverse Proxy)

*   **关键词**：前台接待、保护服务器。

反向代理位于用户和服务器集群之间。对于用户来说，反向代理就像是原始服务器，用户不需要知道后台到底有多少台服务器在工作。

*   **比喻**： 你拨打 10086 客服电话。你不需要知道接电话的是张三还是李四，你只需要拨通这个统一号码（反向代理），它会自动帮你分配一个空闲的客服来服务。

### 主要用途

1.  **负载均衡**： 把大量访问请求分发给后端不同的服务器，防止单台服务器挂掉。
2.  **安全性**： 隐藏真实的服务器 IP。黑客攻击只能打到反向代理上，打不到核心数据服务器。
3.  **加速访问（缓存）**： 代理服务器可以缓存静态资源，让用户访问更快。

## 3. 核心差异对比

| 特性 | 正向代理 (Forward) | 反向代理 (Reverse) |
| :--- | :--- | :--- |
| **代理对象** | 客户端 (用户) | 服务端 (网站/应用) |
| **透明度** | 客户端知道目标服务器是谁 | 客户端不知道具体是哪台后端服务器在提供服务 |
| **位置** | 靠近客户端，通常由用户配置 | 靠近服务端，由网站管理员配置 |
| **典型案例** | VPN、公司内部上网网关 | Nginx、LVS、F5 |

**总结**：正向代理是客户端的“代理人”，替客户端去取东西。反向代理是服务端的“代言人”，替服务端去接客人。
